**日期**: 2024-01-15
**状态**: 设计阶段
**技术栈**: Vue 3

---

## 1. 项目概述

AI产品经理Agent是一个帮助个人开发者和小团队在编码前理清需求思路的工具。用户输入产品想法后，系统通过多个Agent协作，依次进行需求拆解、用户分析、竞品分析、方案设计，最终生成完整的PRD文档。

### 1.1 核心价值

- 避免开发前的无效工作
- 结构化需求分析流程
- 可视化进度展示
- 每步确认，确保方向正确

### 1.2 目标用户

- 个人开发者
- 小型创业团队
- 产品经理
- 想要学习需求分析的开发者

## 2. 产品架构

### 2.1 整体布局

采用左右分栏布局：

```
┌─────────────┬──────────────────────────────────────────────┐
│             │                                              │
│   左侧导航   │              右侧内容区                      │
│             │                                              │
└─────────────┴──────────────────────────────────────────────┘
```

### 2.2 左侧导航结构

| 元素 | 功能 |
|-----|------|
| 🚀 Logo | 品牌展示 |
| ✨ 新建项目 | 需求输入 → Agent管道执行 |
| 📁 项目管理 | 查看/管理已保存的项目 |
| ⚙️ 设置 | 配置（Mock模式/API模式） |
| 👤 用户头像+用户名 | 用户信息（固定底部） |

### 2.3 右侧内容区

根据左侧Tab显示不同内容：
- 新建项目：输入表单 + 管道执行界面
- 项目管理：项目卡片列表
- 设置：配置选项

## 3. 核心功能

### 3.1 新建项目流程

```
输入需求 → 开始执行 → Agent管道执行 → 完成 → 自动保存
```

#### 步骤1：输入需求
- 项目名称输入框
- 需求描述文本域
- "开始执行"按钮

#### 步骤2-6：Agent管道执行
- 5个Agent依次执行
- 每个Agent完成后需要用户确认
- 可查看历史输出
- 可修改当前输出

#### 步骤7：完成
- 显示完成提示
- 项目自动保存到项目管理
- 提供后续操作入口

### 3.2 项目管理

#### 项目列表
- 项目卡片展示（名称、创建时间、进度）
- 查看详情按钮
- 删除按钮

#### 项目详情
- 面包屑导航 + 返回按钮
- 项目概览卡片
- 管道进度展示
- 三个Tab：概览、Agent历史、PRD编辑

### 3.3 Agent管道

| Agent | 输入 | 输出 |
|-------|------|------|
| 拆需求Agent | 用户原始需求 | 项目概述、核心功能列表 |
| 分析用户Agent | 核心功能 | 目标用户、用户画像、痛点 |
| 分析竞品Agent | 核心功能+目标用户 | 竞品对比、差异化优势 |
| 输出方案Agent | 前面所有内容 | 功能详细描述、用户故事 |
| 输出PRDAgent | 前面所有内容 | 完整PRD文档 |

### 3.4 进度可视化

**水平时间轴 + 状态节点**

| 状态 | 图标 | 颜色 | 动画 |
|-----|------|------|------|
| 等待 | ⚪ | 灰色 | 无 |
| 执行中 | 🔵 | 蓝色 | 脉冲 |
| 已完成 | 🟢 | 绿色 | 光晕 |
| 报错 | 🔴 | 红色 | 闪烁 |

### 3.5 Agent输出管理

- 每个Agent的输出持久化保存
- 详情页显示当前/最新输出
- 历史记录页可查看所有Agent的完整输出

### 3.6 PRD编辑

- 在线Markdown编辑器
- 实时预览
- 编辑后保存
- 支持导出

## 6. 技术实现

### 6.1 核心技术

| 技术 | 用途 |
|-----|------|
| Vue 3 | 前端框架 |
| Composition API | 组件逻辑复用 |
| TypeScript | 类型安全 |
| localStorage | 数据持久化 |

### 6.2 UI设计

- **配色方案**: 深色背景 (#0f172a) + 蓝紫渐变 (#3b82f6 → #8b5cf6)
- **科技感元素**: 霓虹光晕、脉冲动画、渐变边框
- **状态过渡**: 平滑的动画过渡

### 6.3 Markdown渲染

- 使用 markdown-it
- 支持代码高亮
- 支持表格、列表等标准语法

### 6.4 配置模式

- **Mock模式**: 预设模板和规则，无需API key
- **API模式**: 真实LLM API调用，支持OpenAI/Anthropic等

## 7. 交互规范

### 7.1 新建项目Tab

1. 用户输入项目名称和需求描述
2. 点击"开始执行"
3. 界面切换到管道执行界面
4. 依次执行5个Agent，每步需要用户确认
5. 全部完成后显示成功提示
6. 项目自动保存

### 7.2 项目管理Tab

1. 显示项目卡片列表
2. 点击"查看"进入项目详情
3. 详情页显示概览、Agent历史、PRD编辑三个Tab
4. 可编辑PRD并导出
5. 点击"返回"或面包屑返回列表

### 7.3 设置Tab

1. 选择Agent执行模式（Mock/API）
2. API模式下配置API key
3. 选择API提供商
